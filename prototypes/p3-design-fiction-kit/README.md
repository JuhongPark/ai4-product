# P3: Design Fiction Workshop Kit

"A Day in the Life of an AI Product Team in 2028" — a 3-hour participatory design fiction workshop for discovering requirements and risks in AI-integrated product teams.

## Kit Contents

| File | Description |
|------|-------------|
| `workshop-protocol.md` | Full facilitator guide: 4-part structure, timing, prompts |
| `scenario-cards.md` | 8 incident scenarios + 4 twist cards (print and cut) |
| `artifact-templates.md` | 4 diegetic artifact templates (post-mortem, retro, policy, news article) |
| `evaluation-form.md` | Individual evaluation form for measuring novelty and quality |

## Run Plan

| Session | Participants | Focus |
|---------|-------------|-------|
| W1 | Product managers (5-7) | PM decision-making with AI |
| W2 | Designers (5-7) | AI in creative workflow |
| W3 | Mixed (PM + designer + engineer, 6-8) | Cross-functional collaboration |

## How to Run

1. Send participant pre-read 2 days before
2. Print scenario cards and artifact templates
3. Follow the 4-part protocol (45 + 60 + 45 + 30 min)
4. Collect all artifacts and evaluation forms
5. Analyze using the post-workshop guide in the protocol

## What It Measures

- Novel requirements discovered (vs. traditional methods)
- Novel risks identified
- Governance rules proposed
- Framework gaps in Selective Complementarity / AADM / Success Patterns
- Participant-rated novelty ("Would traditional methods have found this?")
